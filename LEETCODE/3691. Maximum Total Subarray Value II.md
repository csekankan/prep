# 3691. Maximum Total Subarray Value II

## Problem

Given `nums` of length `n` and integer `k`, pick exactly `k` distinct subarrays `(l, r)`. Each subarray's value = `max(nums[l..r]) - min(nums[l..r])`. Maximize the total value.

---

## Your Approach and Why It Fails

### What you tried

Sort all elements by value (descending). Use a heap to pair the largest element with the smallest element, compute their difference, then count how many subarrays contain both of their original positions. Multiply and subtract from `k`.

### The fundamental flaw: double-counting subarrays

A subarray is a **contiguous range** `[l, r]`. It has exactly **one** value: its own `max - min`. But your approach assigns subarrays to `(max_element, min_element)` pairs. When multiple elements share the same value, **the same subarray gets claimed by multiple pairs**.

### Concrete counterexample

```
nums = [3, 1, 4, 1, 5],  k = 5
Correct answer: 19
Your answer: 20
```

**Step 1:** You pair element `5` (pos 4) with element `1` (pos 3). Subarrays containing both positions 3 and 4: `[0,4], [1,4], [2,4], [3,4]`. All have value 4. Count = 4.

**Step 2:** You pair element `5` (pos 4) with element `1` (pos 1). Subarrays containing both positions 1 and 4: `[0,4], [1,4]`. Both have value 4. Count = 2. You take 1 more.

**But `[0,4]` and `[1,4]` were already counted in Step 1!** These subarrays contain *both* position 1 and position 3 (both have value 1). Your approach sees two different `(max, min)` pairs pointing to the same subarray, and counts it twice.

### Why this is unfixable in your framework

The root cause isn't a bug you can patch — it's that enumerating by `(max_element, min_element)` pairs doesn't give a clean partition of subarrays. You'd need complex inclusion-exclusion to avoid overlaps, which defeats the purpose.

---

## Correct Approach 1: Top-K from Sorted Lists (Heap + Range Queries)

### Step A: The key observation — monotonicity

> For a fixed left endpoint `l`, as you shrink `r` from `n-1` down to `l`, the value `max(l..r) - min(l..r)` is **non-increasing**.

**Why?** When you remove the rightmost element from a subarray:
- The max can only **decrease or stay the same** (you might remove the max)
- The min can only **increase or stay the same** (you might remove the min)
- Therefore `max - min` can only **decrease or stay the same**

**Example with `nums = [4, 2, 5, 1]`, fixed `l = 0`:**

```
[4, 2, 5, 1]  →  max=5, min=1  →  value = 4
[4, 2, 5]     →  max=5, min=2  →  value = 3    (removed min, value dropped)
[4, 2]        →  max=4, min=2  →  value = 2    (removed max, value dropped)
[4]           →  max=4, min=4  →  value = 0

Sequence: 4, 3, 2, 0  ← non-increasing ✓
```

### Step B: This gives us n sorted lists

Each left endpoint `l` produces a non-increasing sequence of values:

```
nums = [4, 2, 5, 1]

l=0: val(0,3)=4, val(0,2)=3, val(0,1)=2, val(0,0)=0  → [4, 3, 2, 0]
l=1: val(1,3)=4, val(1,2)=3, val(1,1)=0                → [4, 3, 0]
l=2: val(2,3)=4, val(2,2)=0                             → [4, 0]
l=3: val(3,3)=0                                         → [0]
```

We want the **top-k values across ALL lists**. This is a classic problem.

### Step C: Top-k from sorted lists = heap merge

This is the same pattern as LC 373 (K Pairs with Smallest Sums) or LC 378 (Kth Smallest in Sorted Matrix):

```
Step 1: Push the FIRST element of each list into a max-heap
        Heap: [(4, l=0, r=3), (4, l=1, r=3), (4, l=2, r=3), (0, l=3, r=3)]

Step 2: Pop the max → (4, l=0, r=3). Add 4 to answer.
        Push the NEXT from list 0 → (3, l=0, r=2)
        Heap: [(4, l=1, r=3), (4, l=2, r=3), (3, l=0, r=2), (0, l=3, r=3)]

Step 3: Pop the max → (4, l=1, r=3). Add 4 to answer.
        Push the NEXT from list 1 → (3, l=1, r=2)
        Heap: [(4, l=2, r=3), (3, l=0, r=2), (3, l=1, r=2), (0, l=3, r=3)]

Step 4: Pop the max → (4, l=2, r=3). Add 4 to answer.
        Push the NEXT from list 2 → (0, l=2, r=2)
        ...
```

For k=3, answer = 4 + 4 + 4 = 12. Correct!

**Why this works:** Because each list is sorted (non-increasing), the next best candidate from any list is always its current head. The heap efficiently finds the global best across all lists.

**Why no double-counting:** Each subarray `[l, r]` appears in exactly one list (the list for left endpoint `l`). So every subarray is counted at most once.

### Step D: The problem — computing val(l, r) fast

Each time we pop `(l, r)` and push `(l, r-1)`, we need to compute:
```
val(l, r-1) = max(nums[l..r-1]) - min(nums[l..r-1])
```

With k pops, we need k range max/min queries. We need these to be fast.

**Naive approach:** Scan from l to r-1 each time → O(n) per query → O(nk) total. Too slow.

**Solution: Sparse Table** — preprocess the array so any range max/min query is O(1).

### Step E: Sparse Table — deep intuition from scratch

#### The problem we're solving

We need to answer thousands of queries like: "What is `max(nums[l..r])`?" for arbitrary `l` and `r`. Scanning the range each time is O(n) per query — too slow.

#### Naive precomputation: why it doesn't scale

You could precompute a 2D table `maxTable[l][r] = max(nums[l..r])` for all pairs. That's O(n^2) space and O(n^2) build time. With n = 50,000, that's 2.5 billion entries — too much.

#### The insight: powers of 2 are enough

Instead of storing the answer for *every* range length, only store ranges whose lengths are **powers of 2**: 1, 2, 4, 8, 16, ...

There are only `log2(n)` such lengths (about 16 for n = 50,000). So the table is only `n * log(n)` entries — very manageable.

```
stMax[i][j] = max of nums[i], nums[i+1], ..., nums[i + 2^j - 1]
              ╰──── starting at index i ────╯  ╰── length = 2^j ──╯
```

#### Building the table: bottom-up, doubling

**j=0 (length 2^0 = 1):** Each element alone. Trivial.
```
stMax[i][0] = nums[i]    for every i
```

**j=1 (length 2^1 = 2):** Max of every pair. Use two length-1 blocks.
```
stMax[i][1] = max(stMax[i][0], stMax[i+1][0])
            = max(nums[i], nums[i+1])
```

**j=2 (length 2^2 = 4):** Max of every group of 4. Use two length-2 blocks.
```
stMax[i][2] = max(stMax[i][1], stMax[i+2][1])
```

**General formula:**
```
stMax[i][j] = max( stMax[i][j-1],  stMax[i + 2^(j-1)][j-1] )
                   ╰─ left half ─╯  ╰──── right half ─────╯
```

A length-2^j range splits into two halves of length 2^(j-1):

```
indices:  i ─────────────────────────── i + 2^j - 1
          |← left half (2^(j-1)) →|← right half (2^(j-1)) →|
          stMax[i][j-1]             stMax[i + 2^(j-1)][j-1]
```

#### Full build example: `nums = [4, 2, 5, 1, 3, 7, 6, 0]`  (n=8)

```
j=0 (len 1):  stMax[i][0] = each element
  index:  0  1  2  3  4  5  6  7
  value:  4  2  5  1  3  7  6  0

j=1 (len 2):  stMax[i][1] = max of 2 adjacent
  stMax[0][1] = max(4, 2) = 4     ← covers [0,1]
  stMax[1][1] = max(2, 5) = 5     ← covers [1,2]
  stMax[2][1] = max(5, 1) = 5     ← covers [2,3]
  stMax[3][1] = max(1, 3) = 3     ← covers [3,4]
  stMax[4][1] = max(3, 7) = 7     ← covers [4,5]
  stMax[5][1] = max(7, 6) = 7     ← covers [5,6]
  stMax[6][1] = max(6, 0) = 6     ← covers [6,7]

j=2 (len 4):  stMax[i][2] = max of 4 adjacent
  stMax[0][2] = max(stMax[0][1], stMax[2][1]) = max(4, 5) = 5    ← covers [0,3]
  stMax[1][2] = max(stMax[1][1], stMax[3][1]) = max(5, 3) = 5    ← covers [1,4]
  stMax[2][2] = max(stMax[2][1], stMax[4][1]) = max(5, 7) = 7    ← covers [2,5]
  stMax[3][2] = max(stMax[3][1], stMax[5][1]) = max(3, 7) = 7    ← covers [3,6]
  stMax[4][2] = max(stMax[4][1], stMax[6][1]) = max(7, 6) = 7    ← covers [4,7]

j=3 (len 8):  stMax[i][3] = max of 8 adjacent
  stMax[0][3] = max(stMax[0][2], stMax[4][2]) = max(5, 7) = 7    ← covers [0,7]
```

**Build code:**
```python
for j in range(1, logn):
    step = 1 << (j - 1)                       # = 2^(j-1), the half-length
    for i in range(n - (1 << j) + 1):          # only valid starting positions
        stMax[i][j] = max(stMax[i][j-1], stMax[i + step][j-1])
```

**Why `range(n - (1 << j) + 1)`?** A range of length `2^j` starting at `i` ends at `i + 2^j - 1`. This must be `< n`, so `i < n - 2^j + 1`.

#### Querying: the overlap trick

Now the clever part. To query `max(nums[l..r])` for *any* l, r:

1. Compute `length = r - l + 1`
2. Find the largest power of 2 that fits: `j = floor(log2(length))`
3. Cover `[l, r]` with TWO blocks of length `2^j`:
   - Block A: starts at `l`, covers `[l, l + 2^j - 1]`
   - Block B: ends at `r`, covers `[r - 2^j + 1, r]`

```
Query: max of [l ........................... r]
              |←── Block A (2^j) ──→|
                         |←── Block B (2^j) ──→|
                         ↑ overlap zone ↑
```

**The blocks might overlap in the middle. That's FINE because `max(A ∪ B) = max(max(A), max(B))`** — counting an element twice in a max doesn't change the result. (This would NOT work for sum.)

**Example:** Query max of `nums[1..6]` in `[4, 2, 5, 1, 3, 7, 6, 0]`
```
length = 6, j = floor(log2(6)) = 2, so blocks of length 4

Block A: stMax[1][2] = max(nums[1..4]) = 5      covers [1,4]
Block B: stMax[3][2] = max(nums[3..6]) = 7      covers [3,6]
                                                  overlap at [3,4]

Answer = max(5, 7) = 7  ✓   (actual max of [2,5,1,3,7,6] = 7)
```

**Query code:**
```python
def queryMax(l, r):
    j = (r - l + 1).bit_length() - 1          # floor(log2(length))
    return max(stMax[l][j],                     # Block A (left-aligned)
               stMax[r - (1 << j) + 1][j])      # Block B (right-aligned)
```

**Why `(r - l + 1).bit_length() - 1`?** In binary, `bit_length()` is the number of bits needed. For example, 6 = `110` in binary, `bit_length() = 3`, so `3 - 1 = 2 = floor(log2(6))`.

#### Why Sparse Table only works for max/min (not sum)

The overlap trick requires an **idempotent** operation — applying it twice to the same element gives the same result:
- `max(5, 5) = 5` ✓ — idempotent
- `min(3, 3) = 3` ✓ — idempotent
- `5 + 5 = 10 ≠ 5` ✗ — NOT idempotent (would double-count overlapping elements)

For sum queries, you'd need a Segment Tree (which splits into disjoint parts, no overlap).

#### Summary

| | Build | Query | Space |
|---|---|---|---|
| Brute force table | O(n^2) | O(1) | O(n^2) |
| **Sparse Table** | **O(n log n)** | **O(1)** | **O(n log n)** |
| Segment Tree | O(n) | O(log n) | O(n) |

### Step E2: Segment Tree — alternative to Sparse Table

#### What a Segment Tree does

Same goal as Sparse Table: answer "what is `max(nums[l..r])`?" quickly. But different tradeoffs:
- Build is faster: O(n) vs O(n log n)
- Query is slower: O(log n) vs O(1)
- Works for **any** operation (sum, max, min, gcd...), not just idempotent ones

#### The structure: a binary tree over the array

Each node covers a contiguous range. Leaves are individual elements. Each parent stores the max (or min) of its two children.

```
nums = [4, 2, 5, 1]

                    [0..3] max=5
                   /            \
           [0..1] max=4       [2..3] max=5
           /      \           /      \
      [0] max=4  [1] max=2  [2] max=5  [3] max=1
```

#### Stored as a flat array (1-indexed)

The tree is stored in an array of size `4n`. Node 1 is the root. For node `i`:
- Left child = `2*i`
- Right child = `2*i + 1`

```
Index:   1    2    3    4    5    6    7
Range: [0,3] [0,1] [2,3] [0,0] [1,1] [2,2] [3,3]
maxv:    5    4     5     4     2     5     1
```

#### Building: bottom-up recursion — O(n)

```python
def build(node, l, r, nums):
    if l == r:                          # leaf
        maxv[node] = nums[l]
        return
    m = (l + r) // 2
    build(2*node, l, m, nums)           # build left child
    build(2*node+1, m+1, r, nums)       # build right child
    maxv[node] = max(maxv[2*node], maxv[2*node+1])   # merge
```

Walkthrough for `nums = [4, 2, 5, 1]`:
```
build(1, 0, 3)
├── build(2, 0, 1)
│   ├── build(4, 0, 0) → maxv[4] = 4        (leaf)
│   ├── build(5, 1, 1) → maxv[5] = 2        (leaf)
│   └── maxv[2] = max(4, 2) = 4             (merge)
├── build(3, 2, 3)
│   ├── build(6, 2, 2) → maxv[6] = 5        (leaf)
│   ├── build(7, 3, 3) → maxv[7] = 1        (leaf)
│   └── maxv[3] = max(5, 1) = 5             (merge)
└── maxv[1] = max(4, 5) = 5                 (merge)
```

#### Querying: divide and conquer — O(log n)

To query `max(nums[ql..qr])`, start at the root and recurse:
- If the node's range is entirely inside `[ql, qr]` → return this node's value (no need to go deeper)
- If the node's range doesn't overlap `[ql, qr]` at all → skip (return -infinity)
- Otherwise → recurse into both children, take the max

```python
def queryMax(node, l, r, ql, qr):
    if ql <= l and r <= qr:          # fully contained
        return maxv[node]
    m = (l + r) // 2
    res = -infinity
    if ql <= m:                       # left child overlaps query
        res = max(res, queryMax(2*node, l, m, ql, qr))
    if qr > m:                        # right child overlaps query
        res = max(res, queryMax(2*node+1, m+1, r, ql, qr))
    return res
```

**Example:** Query `max(nums[1..3])` on `[4, 2, 5, 1]`:

```
queryMax(1, [0,3], ql=1, qr=3)         node covers [0,3], partial overlap
├── queryMax(2, [0,1], ql=1, qr=3)     node covers [0,1], partial overlap
│   ├── queryMax(4, [0,0], ql=1, qr=3) node covers [0,0], NO overlap → skip
│   └── queryMax(5, [1,1], ql=1, qr=3) node covers [1,1], FULLY inside → return 2
│   └── result = 2
├── queryMax(3, [2,3], ql=1, qr=3)     node covers [2,3], FULLY inside → return 5
└── result = max(2, 5) = 5 ✓
```

Only visited 5 nodes out of 7 — and the pruning gets better with larger arrays. At most `2 * log(n)` nodes are visited per query.

#### Why O(log n)? The visual intuition

At each level of the tree, the query range `[ql, qr]` can partially overlap **at most 2 nodes** (one at the left boundary, one at the right). All nodes in between are fully contained and return immediately. So at each of the `log(n)` levels, at most 2 nodes do real work → O(log n) total.

```
Level 0:          [────────── root ──────────]     ← 1 partial node
Level 1:     [──── left ────] [──── right ────]    ← at most 2 partial
Level 2:   [L1] [L2] [R1] [R2]                    ← at most 2 partial
...
Leaves:    a  b  c  d  e  f  g  h                  ← at most 2 partial
```

#### Sparse Table vs Segment Tree — when to use which

| | Sparse Table | Segment Tree |
|---|---|---|
| Query time | **O(1)** | O(log n) |
| Build time | O(n log n) | **O(n)** |
| Space | O(n log n) | **O(n)** |
| Supports updates | No | **Yes** |
| Works for sum | No | **Yes** |
| Works for max/min | Yes | Yes |

**For this problem:** Both work since we only need static range max/min (no updates). Sparse Table has faster queries (O(1) vs O(log n)), so the total complexity is slightly better. But Segment Tree is more versatile and often easier to remember.

#### Segment Tree version of the full solution

```python
class SegTree:
    def __init__(self, nums):
        self.n = len(nums)
        self.maxv = [0] * (4 * self.n)
        self.minv = [0] * (4 * self.n)
        self.build(1, 0, self.n - 1, nums)

    def build(self, node, l, r, nums):
        if l == r:
            self.maxv[node] = self.minv[node] = nums[l]
            return
        m = (l + r) // 2
        self.build(2 * node, l, m, nums)
        self.build(2 * node + 1, m + 1, r, nums)
        self.maxv[node] = max(self.maxv[2*node], self.maxv[2*node+1])
        self.minv[node] = min(self.minv[2*node], self.minv[2*node+1])

    def qmax(self, node, l, r, ql, qr):
        if ql <= l and r <= qr:
            return self.maxv[node]
        m = (l + r) // 2
        res = -(10**18)
        if ql <= m:
            res = max(res, self.qmax(2*node, l, m, ql, qr))
        if qr > m:
            res = max(res, self.qmax(2*node+1, m+1, r, ql, qr))
        return res

    def qmin(self, node, l, r, ql, qr):
        if ql <= l and r <= qr:
            return self.minv[node]
        m = (l + r) // 2
        res = 10**18
        if ql <= m:
            res = min(res, self.qmin(2*node, l, m, ql, qr))
        if qr > m:
            res = min(res, self.qmin(2*node+1, m+1, r, ql, qr))
        return res


class Solution:
    def maxTotalValue(self, nums, k):
        n = len(nums)
        seg = SegTree(nums)

        # Same heap merge as Sparse Table version,
        # just using seg.qmax / seg.qmin instead
        pq = [
            (-(seg.qmax(1,0,n-1,l,n-1) - seg.qmin(1,0,n-1,l,n-1)), l, n-1)
            for l in range(n)
        ]
        heapq.heapify(pq)

        ans = 0
        while k:
            negVal, l, r = heapq.heappop(pq)
            ans -= negVal
            k -= 1
            if r > l:
                val = seg.qmax(1,0,n-1,l,r-1) - seg.qmin(1,0,n-1,l,r-1)
                heapq.heappush(pq, (-val, l, r-1))
        return ans
```

### Step F: Putting it all together

```python
def maxTotalValue(self, nums, k):
    n = len(nums)
    logn = n.bit_length()

    # ---- BUILD SPARSE TABLE ----
    # stMax[i][j] = max of nums[i .. i + 2^j - 1]
    # stMin[i][j] = min of nums[i .. i + 2^j - 1]
    stMax = [[0] * logn for _ in range(n)]
    stMin = [[0] * logn for _ in range(n)]
    for i in range(n):
        stMax[i][0] = stMin[i][0] = nums[i]       # base: length-1 ranges
    for j in range(1, logn):
        step = 1 << (j - 1)                        # half-length
        for i in range(n - (1 << j) + 1):           # valid starting positions
            stMax[i][j] = max(stMax[i][j-1], stMax[i+step][j-1])
            stMin[i][j] = min(stMin[i][j-1], stMin[i+step][j-1])

    # ---- O(1) RANGE QUERIES ----
    def queryMax(l, r):
        j = (r - l + 1).bit_length() - 1
        return max(stMax[l][j], stMax[r - (1 << j) + 1][j])

    def queryMin(l, r):
        j = (r - l + 1).bit_length() - 1
        return min(stMin[l][j], stMin[r - (1 << j) + 1][j])

    # ---- HEAP MERGE OF N SORTED LISTS ----
    # Initialize: push the widest subarray for each left endpoint
    pq = [
        (-(queryMax(l, n-1) - queryMin(l, n-1)), l, n-1)
        for l in range(n)
    ]
    heapq.heapify(pq)        # O(n) heapify

    ans = 0
    while k:
        negVal, l, r = heapq.heappop(pq)     # pop best subarray
        ans -= negVal                          # add its value
        k -= 1
        if r > l:                              # if list not exhausted
            heapq.heappush(pq,                 # push next from same list
                (-(queryMax(l, r-1) - queryMin(l, r-1)), l, r-1))
    return ans
```

### Full walkthrough: `nums = [1, 3, 2], k = 2`

**Sparse Table (max):**
```
j=0: [1, 3, 2]
j=1: max(1,3)=3, max(3,2)=3
```

**Sparse Table (min):**
```
j=0: [1, 3, 2]
j=1: min(1,3)=1, min(3,2)=2
```

**Initialize heap** (one entry per left endpoint):
```
l=0: val(0,2) = queryMax(0,2) - queryMin(0,2) = 3 - 1 = 2  → push (-2, 0, 2)
l=1: val(1,2) = queryMax(1,2) - queryMin(1,2) = 3 - 2 = 1  → push (-1, 1, 2)
l=2: val(2,2) = queryMax(2,2) - queryMin(2,2) = 2 - 2 = 0  → push (0,  2, 2)

Heap: [(-2, 0, 2), (-1, 1, 2), (0, 2, 2)]
```

**Pop 1:** `(-2, 0, 2)` → ans = 2, k = 1
Push next from list 0: val(0, 1) = max(1,3) - min(1,3) = 3 - 1 = 2 → push (-2, 0, 1)

**Pop 2:** `(-2, 0, 1)` → ans = 2 + 2 = 4, k = 0

**Answer: 4** ✓

### How to think of this approach

Ask yourself: **"Can I order subarrays by value without enumerating all O(n^2) of them?"**

The answer is yes — by fixing one endpoint and observing the monotonicity. This is a pattern that appears whenever:
- You need the top-k from a structured space
- The space can be decomposed into sorted sequences
- Each sequence can be lazily evaluated (you only compute the next element when needed)

Similar problems: k-th smallest pair sum, k-th smallest element in a sorted matrix.

## Correct Approach 2: Binary Search + Sliding Window

### The idea

Instead of finding the top-k subarrays directly, ask:

> "What is the threshold value `v` such that exactly `k` subarrays have value >= `v`?"

Then the answer = (sum of all values > v) + v * (remaining slots).

### Three building blocks

#### Block 1: Total sum via monotone stacks — O(n)

Compute `T = sum of (max - min)` over ALL subarrays.

For each element `nums[i]`, use the "previous greater / next greater" monotone stack technique to count how many subarrays it's the max of (call it `f_max(i)`), and similarly for min. Then:

```
sum_of_all_max = sum(nums[i] * f_max(i))
sum_of_all_min = sum(nums[i] * f_min(i))
T = sum_of_all_max - sum_of_all_min
```

This is the classic "sum of subarray ranges" pattern (LC 2104).

#### Block 2: Count and sum subarrays with value <= v — O(n) sliding window

Maintain a window `[L, r]` where `max - min <= v`. For each `r`, find the smallest valid `L`. Use two monotone deques (one for max, one for min) to track the window's max and min.

**The non-obvious part:** We also need `sum of (max - min)` for all valid subarrays, not just the count. We maintain running sums:

```
s_max = sum of max(l..r) for all l in [L, r]
s_min = sum of min(l..r) for all l in [L, r]
```

These are updated incrementally:
- **When pushing `r`:** Elements popped from the max-deque had a smaller max; now `nums[r]` takes over for those left endpoints. Adjust `s_max` by `(nums[r] - old_value) * count`.
- **When advancing `L`:** Remove the contribution of left endpoint `L`. Since `max(L..r) = deque_front`, just subtract it.

#### Block 3: Binary search on the threshold — O(log V) iterations

```
Binary search for largest v where count(value >= v) >= k
    count(value >= v) = total_subarrays - count(value <= v-1)

Answer = sum(values > v*) + v* * (k - count(values > v*))
       = (T - sum_leq(v*)) + v* * (k - (total - count_leq(v*)))
```

### Complete solution

```python
def maxTotalValue(self, nums, k):
    n = len(nums)
    total_sub = n * (n + 1) // 2

    def stack_sum(pop_cond):
        res, stk = 0, []
        for i in range(n):
            while stk and pop_cond(nums[stk[-1]], nums[i]):
                j = stk.pop()
                left = stk[-1] if stk else -1
                res += nums[j] * (j - left) * (i - j)
            stk.append(i)
        while stk:
            j = stk.pop()
            left = stk[-1] if stk else -1
            res += nums[j] * (j - left) * (n - j)
        return res

    T = stack_sum(lambda a, b: a <= b) - stack_sum(lambda a, b: a >= b)

    def sw(v):
        mx_dq, mn_dq = deque(), deque()
        s_mx = s_mn = cnt = sm = L = 0
        for r in range(n):
            while mx_dq and nums[mx_dq[-1]] <= nums[r]:
                p = mx_dq.pop()
                pv = mx_dq[-1] if mx_dq else L - 1
                s_mx += (nums[r] - nums[p]) * (p - pv)
            mx_dq.append(r)
            s_mx += nums[r]

            while mn_dq and nums[mn_dq[-1]] >= nums[r]:
                p = mn_dq.pop()
                pv = mn_dq[-1] if mn_dq else L - 1
                s_mn += (nums[r] - nums[p]) * (p - pv)
            mn_dq.append(r)
            s_mn += nums[r]

            while mx_dq and mn_dq and nums[mx_dq[0]] - nums[mn_dq[0]] > v:
                s_mx -= nums[mx_dq[0]]
                s_mn -= nums[mn_dq[0]]
                if mx_dq[0] == L: mx_dq.popleft()
                if mn_dq[0] == L: mn_dq.popleft()
                L += 1

            cnt += r - L + 1
            sm += s_mx - s_mn
        return cnt, sm

    lo, hi = 0, max(nums) - min(nums)
    while lo < hi:
        mid = (lo + hi + 1) // 2
        c, _ = sw(mid - 1)
        if total_sub - c >= k:
            lo = mid
        else:
            hi = mid - 1

    cl, sl = sw(lo)
    c = total_sub - cl
    s = T - sl
    return s + lo * (k - c)
```

## How to Arrive at These Solutions During an Interview

### Step 1: Recognize the structure

"I need top-k subarray values. There are O(n^2) subarrays. I can't enumerate them all when n = 50,000."

### Step 2: Ask — can I order them without listing them all?

**Binary search approach:** "Instead of finding the top k, what if I could just count how many subarrays are above a threshold? If that count function is cheap, I can binary search."

### Step 3: Choose based on what you know

- If you're comfortable with sparse tables or segment trees, the heap approach is more intuitive.
- If you're comfortable with monotone deques and sliding windows, the binary search approach avoids any extra data structures.

### Your mistake in a nutshell

You tried to decompose subarrays by their (max_element, min_element) pair. But subarrays are defined by their **positions** `(l, r)`, not by which elements happen to be the max and min. Multiple element pairs can point to the same subarray when duplicates exist. The correct decompositions are:
- **By left endpoint** (heap approach) — clean partition, monotone property
- **By value threshold** (binary search approach) — count/sum split

### Bug Pattern 1: Copy-paste errors between max/min variants

### Bug Pattern 2: Forgetting to pass `node` in recursive calls

You wrote this 3 times:
```python

```

**Why it matters:** The `node` parameter tells the function which slot in the flat array to read. Without it, the function doesn't know where it is in the tree.

**Memory aid:** Recursive calls always have 3 pairs:
```
self.queryMax(  2*node,    l,   m,   ql, qr)   ← left child
self.queryMax(  2*node+1,  m+1, r,   ql, qr)   ← right child
                ↑ child     ↑ child's range  ↑ query (unchanged)
```

### Bug Pattern 3: Off-by-one with `m` and `m+1`

You initially wrote `self.build(2*node+1, m, r)` instead of `m+1, r`. This causes infinite recursion because both children cover index `m`.

**Rule:** Left child = `[l, m]`. Right child = `[m+1, r]`. The `+1` ensures no overlap and guarantees progress toward base case.

Same error in query: `qr >= m` should be `qr > m`. The right child starts at `m+1`, so the query must extend past `m` to overlap it.

### Bug Pattern 4: 0-indexed vs 1-indexed root

You used `tree.build(0, ...)` and `tree.queryMax(0, ...)`. With `2*node` / `2*node+1` children, root must be **node 1**:
- Node 0 → left child = 2*0 = 0 (itself!) → infinite recursion
- Node 1 → left child = 2, right child = 3 ✓

**Rule:** Always start with `node=1` when using the `2*node` convention.

### Bug Pattern 5: Missing guard before pushing to heap

After popping `(l, r)` and doing `r -= 1`, you must check `r >= l` before querying. A single-element subarray (`l == r`) has no "next" in the list.

```python

# CORRECT

    r -= 1
    heapq.heappush(heap, (..., l, r))
```

### Bug Pattern 6: Duplicate heap pushes

You wrote `heappush` twice for the same entry. This doubles every value after the first pop. Always push exactly once.

### Bug Pattern 7: Forgetting to call `build` in `__init__`

The Segment Tree constructor allocated arrays but never built the tree. Either call `self.build(1, 0, self.n - 1)` inside `__init__`, or remember to call it externally.

## Quick-Reference: Segment Tree Template

```python
class SegTree:
    def __init__(self, arr):
        self.n = len(arr)
        self.maxv = [0] * (4 * self.n)
        self.minv = [0] * (4 * self.n)
        self.nums = arr
        self.build(1, 0, self.n - 1)           # ← don't forget to call

    def build(self, nd, l, r):
        if l == r:
            self.maxv[nd] = self.minv[nd] = self.nums[l]
            return
        m = (l + r) // 2
        self.build(2*nd, l, m)                  # ← left child
        self.build(2*nd+1, m+1, r)              # ← m+1, not m
        self.maxv[nd] = max(self.maxv[2*nd], self.maxv[2*nd+1])
        self.minv[nd] = min(self.minv[2*nd], self.minv[2*nd+1])
        #                 ^^^ min, not max     ^^^ minv, not maxv

    def qmax(self, nd, l, r, ql, qr):
        if ql <= l and r <= qr:
            return self.maxv[nd]
        m = (l + r) // 2
        res = float('-inf')                     # ← -inf for max
        if ql <= m:
            res = max(res, self.qmax(2*nd, l, m, ql, qr))
        if qr > m:                              # ← strict >, not >=
            res = max(res, self.qmax(2*nd+1, m+1, r, ql, qr))
        return res                              # pass nd ↑ in every call

    def qmin(self, nd, l, r, ql, qr):          # 4 changes from qmax:
        if ql <= l and r <= qr:
            return self.minv[nd]                # 1. minv not maxv
        m = (l + r) // 2
        res = float('inf')                      # 2. +inf not -inf
        if ql <= m:
            res = min(res, self.qmin(2*nd, l, m, ql, qr))
        if qr > m:                              # 3. min() not max()
            res = min(res, self.qmin(2*nd+1, m+1, r, ql, qr))
        return res                              # 4. qmin not qmax
```

### Your final correct solution (Segment Tree + Heap Merge)

```python
class Solution:
    def maxTotalValue(self, nums, k):
        n = len(nums)
        tree = SegTree(nums)
        heap = []
        for l in range(n):
            mx = tree.qmax(1, 0, n-1, l, n-1)
            mn = tree.qmin(1, 0, n-1, l, n-1)
            heapq.heappush(heap, (-(mx - mn), l, n - 1))
        result = 0
        while heap and k > 0:
            k -= 1
            val, l, r = heapq.heappop(heap)
            result -= val
            if r > l:                           # ← guard!
                r -= 1
                mx = tree.qmax(1, 0, n-1, l, r)
                mn = tree.qmin(1, 0, n-1, l, r)
                heapq.heappush(heap, (-(mx - mn), l, r))  # ← push ONCE
        return result
```
